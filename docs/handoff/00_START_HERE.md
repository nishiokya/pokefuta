## 3. データモデル是正（②の中身・全PR共通の前提）
README §2.5 と同一。Claude Code はこれを**全ブランチで遵守**すること。
- **全国マンホール総数 = 470**。コレクションの分母は常に 470。
- **訪問 = 撮影 = 1スタンプ**。「訪問したが未撮影」という個人ギャップは存在しない（`52/80` のような訪問≠撮影表現は誤り）。個人コレクション = 訪問数 = 撮影数 = 例 `80/470`（完成率≈17%）。
- **写真投稿KPI（写真0枚 / 一番乗り）は「未訪問マンホール」の話**。写真を増やす導線は**詳細ページ／探す（未訪問側）**の `PhotoPrompt` が担う。
- **マイ旅（過去の訪問）に「写真を追加」CTAは置かない**（既に撮影済み）。
- ②でスタンプ帳の表示を触るのが不安なら、**スタンプ帳の数値訂正は④に回し**、②は詳細・マイ旅・地図など他画面のみ470化する。
