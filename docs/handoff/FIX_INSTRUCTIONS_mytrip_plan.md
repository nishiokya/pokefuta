## 何が劣後しているか（現状の問題）
1. **達成率カードが巨大なヒーロー**: `POKEFUTA PASSPORT / 全国80/470・都道府県10/47・ポケモン104/534` が画面上部1/3を占有。達成率はCランク（最劣後）施策なのに最も目立つ。
2. **収集の進捗が主役化されていない**: 「集めたポケふたが何枚か（n/470）」「今月いくつ増えたか」が見えない。
3. **タイムラインが写真中心の旅日記になっていない**（同一レコードの重複も発生）。

> ★データモデル（README §2.5）: **訪問=撮影**。マイ旅の各レコードは必ず本人写真を持つ。**「写真がない訪問」は存在しないので、マイ旅に「写真を追加」CTAは置かない**（写真を増やす導線は未訪問マンホール側＝詳細/探す）。分母は **470**。
4. **CTAが重複・冗長**（訪問を記録／スタンプ帳が上下で重複）。

## あるべき構造（`MyTrip` 準拠）
画面は上から **①細ヘッダー（写真図鑑が主役）→ ②月グルーピングの旅日記タイムライン** の2層だけ。

### ① 細ヘッダー：写真図鑑を主役、達成率は二次行へ降格
PASSPORTの巨大カードを廃し、**コンパクトな1枚カード**に置換（`reference` の slim header）。
- ラベル `PHOTO DEX`(amber-d, micro)。
- **左 = 主役指標**: 「集めたポケふた」`80/470`（Outfit 800/26px）。
- **右 = モメンタム指標**: 「今月」`+6`（terracotta・Outfit 800/26px）＋ `今年 24件`。
- メーター: `linear-gradient(90deg,#e2a015,#bf5640)`、`height 9 / radius 999`。
- **達成率は最下部の細い二次行**に降格: `達成率 全国 80/470 · 都道府県 10/47 ›`（11px・inkFaint、上に `--line-soft` の区切り）。タップで展開（ポケモン達成率等はここを開いた中へ）。**決して最上部の主役にしない。**

### ② 旅日記タイムライン：月でグルーピング、写真中心
- **月ヘッダー** `MT_MonthHead`: `2026年5月` + `5件` チップ + 罫線。`white-space:nowrap`。
- **写真あり = 横長フォトカード** `MT_PhotoCard`（2カラムグリッド）:
  - 写真エリア `height 150`、下方向の暗グラデーション、**写真の上にタイトル＋日付を白文字オーバーレイ**。
  - 写真下に希少性ピル（塗りティント peach/lav/mint、`fontSize 10`）。
- **写真ゼロ訪問の行は作らない**（訪問=撮影のため存在しない）。マイ旅は純粋な旅日記。写真を増やす導線は未訪問マンホール（詳細/探す）側の `PhotoPrompt` が担う。

> 重複レコード（名古屋×3）は **dedup**（同一 manholeId × 訪問日でユニーク化）。
