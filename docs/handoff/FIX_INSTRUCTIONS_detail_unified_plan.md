# 修正指示書（plan用）: 詳細ページを「1テンプレート」に統合し、全画面トグルを廃止

> 対象: Claude Code（plan モードにそのまま貼る）
> 正となるデザイン: `reference/screens-Unified.jsx` の `U_Detail`（プレビュー: セクション「統合版 詳細ページ」）。
> トークン: `README.md §7`。

---

## 問題（現状の実装）
詳細ページが **2テンプレートに分裂**していて、「あとで」を押すと**ページ全体が旧テンプレートに書き換わる**。
- 状態A = 新 PhotoPrompt 主役（一番乗り＋図鑑進捗）
- 状態B = 旧詳細（NEXT STAMP プレースホルダ／注目タグの縦長大行／写真追加・経路案内／X・LINE・共有）

**これは設計ミス。** PhotoPrompt は「ページの別バージョン」ではなく「**ページ内の1セクション**」であるべき。
「あとで」で全画面が切り替わるのはUXとして破綻している（同じポケふたなのに別ページに見える）。

## ゴール
**詳細ページは1テンプレートだけ。** PhotoPrompt を最上段に置き、旧ページ固有の情報（建物・目印／登場ポケモン／場所／共有）は **その下に常設**マージする。「あとで」による全画面トグルは廃止。

---

## あるべき構造（上から・`U_Detail` 準拠）

1. **PhotoPrompt（最上段・主役）** — 既存の empty/posted 出し分けをそのまま。
   - ヒーロー（点線枠＋大 `0`＋一番乗りチャンス badge、右上に `未訪問`）
   - ロケーション＋タイトル＋一言説明（「オタマロ・ガマガル・ガマゲロゲが描かれた…」）
   - 統合カード（`0人 › #1` リボン ＋「撮ると写真図鑑も埋まる `0→1 / 5` ＋ +1 シリーズ進捗」）
   - 希少性ピル（塗りティント）
   - **主CTA `一番乗りで投稿する`（terracotta）**
2. **区切り線**（`--line` 1px）
3. **建物・目印** — アイコン＋ラベル「建物・目印」＋「下呂市観光交流センター『湯めぐり館』」のカード。
4. **登場ポケモン** — 節見出し＋ポケモンチップ（白地/line枠/丸アイコン）。
5. **場所** — 節見出し＋地図カード（マップ＋住所＋`経路案内`グリーン）。
6. **共有** — 節見出し＋`X で共有`(黒) / `LINEで共有`(緑#06c755) / `共有`(白)。SNS拡散はKPIなので**常設**で下部に置く。

> ※旧ページの「写真追加」ボタンは、最上段 PhotoPrompt の主CTAに集約済み。重複させない。
> ※旧ページの「NEXT STAMP」プレースホルダ・ヒーローは廃止（PhotoPrompt のヒーローに一本化）。

---

## 「あとで」の扱い（重要）
- **詳細ページでは「あとで」ボタンを削除**する。プロンプトは本文の主役で、**押さずにスクロールすれば実質「あとで」**＝逃げ道は足りている。
- `dismissed` フラグによる**テンプレート差し替えロジックは丸ごと削除**。詳細ページに「PhotoPrompt版/旧版」の分岐を残さない。
- （ホーム/マイ旅などの**差し込みバナー**で「あとで」を残す場合のみ）挙動は collapse（その場で畳む）＋ id 変化でリセット。**全画面トグル・dismiss全消しは禁止。**

## 受け入れ基準（Done定義）
- [ ] 詳細ページのテンプレートが**1つ**。「あとで」で別レイアウトに切り替わらない。
- [ ] PhotoPrompt が最上段、その下に **建物・目印／登場ポケモン／場所／共有が常に表示**される（スクロールで全部見える）。

- [ ] 「写真追加」CTA が最上段に1つだけ（重複なし）。
- [ ] ピル＝塗りティント、字体・余白・影が §7 準拠。
