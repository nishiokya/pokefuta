## 問題（現状の実装）
スタンプ帳が **訪問済みマンホールを日付順に並べたグリッド**（円形アイコン＋地名＋`2026/3` …）になっており、**マイ旅（時系列ログ）とほぼ同じ**。結果として:
- 都道府県収集が見えない / ポケモン収集が見えない / 次に集める対象が見えない / コンプリート欲が刺激されない。

スタンプ帳の本質は「旅日記」ではなく **コレクションブック**（御朱印帳・道の駅スタンプラリー・マンホールカードに近い）。**「行った・撮った・集めた・残した」のうち "集めた" を主役**にする画面。

## スタンプ帳の構造（`Stampbook` 準拠・上から）

0. **コレクション軸セグメント**: `ぜんぶ / 都道府県 / ポケモン / 特徴`（白タブ＝アクティブ）。この画面がコレクションブックであることを最初に宣言。

> ### ★v2 追記: 「図鑑の壁（集めたスタンプ）」で密度を回復する
> 初回実装は仕様どおりだが、**バー＋数字だけだと“ダッシュボード”化して収集の充実感（御朱印帳的な「集まった現物の壁」）が薄い**。これを補うため、写真図鑑サマリーの**直下に「集めたスタンプ」グリッド**を必ず置く（`reference/screens-Stampbook.jsx` の `Stampbook2` / `SB_DexWall`）。
> - **集めた現物（マンホール画像）を 8 列グリッドでぎっしり**表示（円形・本番は実画像、プレースホルダは塗りパターン）。日付は出さない（=マイ旅化しない。図鑑/地域順）。
> - 末尾に**点線の空き枠を数個**置き、`📷`付きで「次に狙える近くのポケふた」を示す（再訪・新規訪問の動機）。見出し右に `すべて見る 80 ›`。
> - 写真図鑑サマリー自体は**コンパクト化**（`80/470` と `17%・あと390` を1行ずつ、余白を詰める）。PC幅で間延びさせない。
> - ポケモントークンは**空（◓）にしない**。集めた現物の塗り（完成＝緑リング＋カラー、未完＝淡いグレースケール）で“持っている感”を出す。
> - 効果: 「集めた手応え（壁）」＋「あと少しで埋まる（空き枠）」が一目で伝わり、収集体験が戻る。**これを推奨採用とする。**

1. **写真図鑑（収集を強く）**: ラベル `PHOTO DEX` ＋「写真図鑑コンプリート」＋ 右に大きく **`80/470` 集めたポケふた**。メーター（`linear-gradient(90deg,#e2a015,#bf5640)`、17%）。下に3分割で `完成率 17% / 今月 +6(terracotta) / あと 390`。
   - ★データモデル（README §2.5）: **訪問=撮影**。分母は **470**。「訪問数80のうち撮影52」のような訪問≠撮影の表現は誤り。集めた数＝訪問数＝撮影数。

2. **コンプリート目前（再訪動機・横スクロール）**: trophy見出し＋`すべて見る ›`。カード列: `あと1枚 岐阜県完成 / あと2枚 ロコン完成 / あと1枚 道の駅制覇`。各カードに `📷 次を撮る`。**再訪問動機を最も生む要素**なので上部に置く。

3. **都道府県コレクション（中心コンテンツ）**: カード内に横バーのリスト。`北海道 42/50`・`岐阜県 14/15`・`鹿児島県 18/26`・`愛知県 9/9(✓緑)`。完成は green、未完は amber→terracotta グラデ。フッターに `あと3県でコンプリート`。右上 `47都道府県 ›`。

4. **ポケモンコレクション（ポケふたならでは）**: 円形トークン横並び。完成はリングを green＋「コンプ」、未完は `5/18` 等。`イーブイ 8/8 / ロコン 27/27 / ピカチュウ 5/18 / ヤドン 3/11`。右上 `534匹 ›`。

5. **特徴コレクション（旅好きに刺さる）**: ピル群 `道の駅 8/9（目前=gold強調）/ 世界遺産 3/7 / 離島 2/12 / 駅前 11/14 / 海沿い 6/20`。右上 `すべて ›`。

> セグメントで `都道府県/ポケモン/特徴` を選ぶと、その軸の**全件一覧**（コンプリート済み・未到達・目前でソート）に展開する想定。`ぜんぶ` は上記サマリー。
