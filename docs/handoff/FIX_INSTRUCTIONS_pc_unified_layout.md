# 修正指示書（plan用）: ログイン後PC画面を「1つの骨格」に統一する（横幅バラバラの根治）

> 対象: Claude Code（plan モードにそのまま貼る）

> トークン: `README.md §7` ＋ 本書の共通グリッド表。

---

## なぜ今これなのか（経緯）
PC幅のログイン後画面で「横幅がバラバラ」。原因は **仕様の二重化**だった:

- `README §4.3`（旧）: `1fr 372px / gap 24 / padding 28` → レール **372**・ガター **28**・gap **24**
- `reference/screens-PC.jsx`（実体）: `PC_W 1120 / PC_PAD 32 / rail 360 / gap 28`

数値が全項目で食い違い、参照する資料によって出来上がる幅が変わる。**画面ごとの小さなPRを積んでも収束しない**ため、関連PRはすべてクローズし、**1ブランチ `feat/pc-unified-layout` で一括統一**する。`README §4.3` は既に reference の数値へ更新済み（372→360 / 24→28 / 28→32）。**今後 372/24/28 の旧値は使わない。**

---

## ゴール
**ログイン後のPC画面は全ページで1つの骨格 `PC_Shell` を共有する。** フレーム幅・左右ガター・本文カラム・カラム間・右レール幅は全ページで完全一致。ナビ・パンくず・本文・レールの**左右の端がページ間でピクセル一致**する。

## 共通グリッド（全PCページ固定・`PC_Shell`）

| トークン | 値 | 用途 |
|---|---|---|
| `PC_W`（フレーム幅） | **1120px** | ページ枠。中央寄せ |
| `PC_PAD`（左右ガター） | **32px** | ナビ・パンくず・本文すべて同じ左右端 |
| 本文カラム | **`minmax(0,1fr)` ≈ 668px** | 可変。`min-width:0` 必須（はみ出し防止） |
| `PC_GAP`（カラム間） | **28px** | |
| `PC_RAIL`（右レール） | **360px** | `position:sticky; top:20px` |
| 本文ブロック余白 | `padding:20px 32px 30px` ／ 本文内 `gap 20` ／ レール内 `gap 14` | |

```
|← 32 →|        本文カラム 1fr (≈668)        |← 28 →|   右レール 360 (sticky)   |← 32 →|
|------------------------------ フレーム 1120 ------------------------------------------|
```

検算: `32 + 668 + 28 + 360 + 32 = 1120`。整列ガイドの基準線は `x=32 / 700 / 728 / 1088`。

## 実装（`PC_Shell` を1つ作り、全ページがそれを使う）

### 1. シェルを単一コンポーネント化
`reference/screens-PC.jsx` の `PC_Shell({ active, crumbs, children, rail })` をコードベースのレイアウト規約で1つだけ作る。中身:
- 上段: `PC_TopNav`（ロゴ＋`探す`/`スタンプ帳`/`まい旅`＋検索＋ユーザー）。`padding: 14px 32px`。
- パンくず（任意）: `padding` は無し、グリッドと同じ左端 `32`。
- 本文グリッド: `display:grid; grid-template-columns: minmax(0,1fr) 360px; gap:28px; align-items:start;`
  - 左 = `children`（本文・`min-width:0`・`gap 20`）
  - 右 = `rail`（`position:sticky; top:20px`・`gap 14`）

### 2. 各ページを `PC_Shell` に載せ替える（横幅を独自指定しない）
**どのページも独自の幅・ガター・グリッドを書かない。** 必ず `PC_Shell` の `children` と `rail` に流し込む。

| ページ | `active` | 本文（children） | 右レール（rail） |
|---|---|---|---|
| スタンプ帳ホーム | `stamp` | 「あと2枚でコンプ」＋ コレクション**5列**グリッド（円 96px） | 写真図鑑カード `80/470`＋メーター＋今月/達成率＋`撮りに行く` |
| 詳細・写真0枚 | `search` | ヒーロー(340)＋タイトル＋希少性ピル＋地図 | PhotoPrompt（`0人›#1`＋図鑑進捗＋`一番乗りで投稿する`） |
| 詳細・写真あり | `search` | ギャラリー(340)＋サムネ＋タイトル＋ピル | PhotoPrompt（`0/1図鑑`＋`あなたの1枚を加える`） |
| まい旅 | `mytrip` | 時系列の旅日記（README/`FIX_INSTRUCTIONS_mytrip_plan.md` 準拠） | 旅の集計サマリ（写真追加CTAは置かない＝撮影済み） |
| 探す（一覧/地図） | `search` | 一覧・地図本体 | 絞り込み／選択中ピンの要約 |

> まい旅・探すのPC版デザインはまだ起こしていないが、**幅の規約は同じ**: 必ず `PC_Shell`（1120 / 32 / 1fr+28+360）。中身の出し分けは各 `FIX_INSTRUCTIONS_*` に従う。

### 3. 旧PCコードの撤去
- ページ固有の `max-width: 1040`、`372px`、`gap:24`、`padding:…28` などの**ハードコードを全削除**。幅は `PC_Shell` のみが持つ。
- 詳細ページの「あとで全画面トグル」分岐は `FIX_INSTRUCTIONS_detail_unified_plan.md` に従い撤去（PCでも1テンプレート）。
