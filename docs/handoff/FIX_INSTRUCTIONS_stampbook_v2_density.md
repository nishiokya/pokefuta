# 追加実装指示: スタンプ帳に「図鑑の壁」で密度を足す（v1→v2 差分のみ）

> 対象: Claude Code（plan にそのまま貼る）
> 前提: スタンプ帳のコレクションブック化（v1）は実装済み。本書は**そこに3点足すだけの差分**。
> 正となるデザイン: `docs/handoff/reference/screens-Stampbook.jsx` の **`Stampbook2` / `SB_DexWall` / `SB_PokeToken`**（プレビュー `reference/ポケふた 写真投稿導線 3案.html` の「スタンプ帳 v2 ★推奨」）。
> データモデル/トークン: README §2.5 / §7。

## なぜ（背景）
現状のスタンプ帳はバー＋数字だけで“ダッシュボード化”し、御朱印帳的な「集めた現物の壁」の充実感が無い。下記3点で密度を回復する。**構造（セグメント/コンプリート目前/都道府県/ポケモン/特徴）は変えない。**

## やること（3点だけ）

### 1. ★「集めたスタンプ」グリッドを新設（最重要）
写真図鑑サマリーの**直下**に、カードで「集めたスタンプ」セクションを追加する。
- 見出し: 左 `🖼/スタンプアイコン 集めたスタンプ`、右 `すべて見る 80 ›`。
- 本体: **8列グリッド**（`grid-template-columns: repeat(8,1fr)`, gap 7）で円形セルをぎっしり。
  - **集めた現物**＝塗り（本番は投稿写真の円形サムネ。無ければマンホール風の斜めストライプ `repeating-linear-gradient(45deg,#d9cdb2 0 4px,#cfc2a3 4px 8px)`＋内側に細い白リング）。
  - 末尾に**点線の空き枠を数個**。うち2〜3個は terracotta 点線＋中央に `📷`＝「次に狙える近くのポケふた」。
- フッターに凡例1行: `点線＝次に狙える近くのポケふた`。
- **日付は出さない**（マイ旅化しない。図鑑/地域順）。表示は最大24〜40個程度＋`すべて見る`で全件へ。

### 2. 写真図鑑サマリーをコンパクト化
現状の大きい3分割（`17% / +0 / 390` を縦に大きく）をやめ、**1カードに圧縮**:
- 左に `80/470`（Outfit 800/22px）、右に `17%` ＋ 小さく `完成率・あと390`。その下にメーター（17%）。
- 余白を詰める（PC幅で間延びさせない）。`+0 今月` は小さく副次表示でよい。

### 3. ポケモントークンを「現物」にする
現状の空の `◓` をやめ、**塗り**にする（`SB_PokeToken` 準拠）:
- 円の中身＝マンホール風ストライプ。**完成**＝緑リング＋フルカラー＋ラベル`コンプ`、**未完**＝淡いグレースケール（opacity 0.45 / grayscale）＋`5/18`。

## 受け入れ基準
- [ ] 写真図鑑の直下に「集めたスタンプ」8列グリッドが出る（現物ぎっしり＋点線の空き枠＋📷）。
- [ ] 写真図鑑サマリーがコンパクト（大きい3分割ではない）。
- [ ] ポケモンが塗り（完成=緑/未完=淡色）で、空の◓ではない。
- [ ] 構造（セグメント・コンプリート目前・都道府県・ポケモン・特徴）はv1のまま。
- [ ] 日付グリッドは出さない。数値は 470 / 訪問=撮影（README §2.5）。
